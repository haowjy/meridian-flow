Architecture brief

This document describes the high-level structure of the Atlas platform, covering service boundaries, data flow, and the key decisions behind each layer. It is intended as a starting point for new contributors and a reference during review.

Summary
Atlas uses an event-driven architecture with three core services: ingestion, processing, and delivery. Each service owns its data store and communicates through a shared message bus.
